11.5 Kommunikationsmodell 



Hier klicken, um das Bild zu vergrößern

Abbildung 11.4    Kommunikationsmodell von Sockets

In der Abbildung 11.4 sehen Sie das Kommunikationsmodell, auf dem die Sockets aufgebaut sind. Damit zwei Prozesse über eine TCP-Verbindung miteinander kommunizieren können, muss erst einmal ein Socket angelegt oder geöffnet werden. Anschließend kann über dieses Socket eine Verbindung zum anderen Prozess aufgebaut werden, der auf diese Verbindung wartet. Wenn die Verbindung erst einmal steht, können mit normalen Standardfunktionen Daten an das Socket übergeben werden, die dann über die TCP-Verbindung an den anderen Prozess übertragen werden. Der andere Prozess kann wiederum diese Daten vom Socket lesen. Das Prinzip stellt ja für Sie nichts Neues mehr dar, sofern Sie das Kapitel »Interprozesskommunikation« gelesen haben. Es ist also auch nichts anderes als eine Kommunikation zwischen zwei Prozessen, die nicht vom Standardort des Rechners abhängig sind.
